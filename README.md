# lunar-gateway

**Stateless edge gateway for LunarAST ecosystem data distribution.**

`lunar-gateway` is a Cloudflare Workers Wasm service that provides secure, cached, and parameterized access to `lunar-map.json` — the global topology generated by `lunar map`. It is the production-grade replacement for `lunar-serve`, offering global CDN distribution, Ed25519-JWT authentication, and edge-native caching.

---

## Quick Start

### 1. Prerequisites

- Cloudflare account with Workers and R2 enabled
- [Wrangler](https://developers.cloudflare.com/workers/wrangler/) CLI installed
- A `lunar-map.json` file generated by `lunar map`

### 2. Configure R2 Bucket

Create an R2 bucket (e.g., `lunar-ast-yourorg`) and upload your topology data:

```bash
wrangler r2 bucket create lunar-ast-yourorg
wrangler r2 object put lunar-ast-yourorg/lunar-map.json -f /path/to/lunar-map.json
```

### 3. Deploy

Edit `wrangler.toml` to match your bucket name and add your public keys:

```toml
name = "lunar-gateway"
main = "src/lib.rs"
compatibility_date = "2026-06-08"

[[r2_buckets]]
binding = "LUNAR_BUCKET"
bucket_name = "lunar-ast-yourorg"

[vars]
LUNAR_PUBLIC_KEYS = "{ \"your-project\": \"<public-key-hex>\" }"
```

Then deploy:

```bash
wrangler deploy
```

## Endpoints

| Endpoint | Auth | Description |
|:---|:---|:---|
| `GET /healthz` | None | Health check |
| `GET /lunar-map.json` | None | Full topology JSON |
| `GET /lunar-map.md` | None | Markdown rendering with optional filters (`?summary`, `?scope`, `?status`, `?path`, `?style`) |
| `GET /private/lunar-map.md` | JWT | Same as above, requires valid Ed25519 JWT for private projects |

### Filter Parameters

| Parameter | Example | Description |
|:---|:---|:---|
| `?summary=true` | `?summary=true` | Ecosystem summary (~200 tokens) |
| `?scope=<project>` | `?scope=auth-service` | Single project view |
| `?status=<status>` | `?status=orphaned` | Filter alignments by status |
| `?path=<path>` | `?path=/auth` | Filter alignments by path |
| `?style=mermaid` | `?style=mermaid` | Mermaid topology diagram |

## Authentication

`lunar-gateway` uses Ed25519 (EdDSA) for JWT verification, as specified in Section 8.3 of the Ecosystem Mother Spec.

### 1. Generate a Key Pair

```bash
lunar keygen my-project
```

This creates a private key at `~/.lunar/keys/my-project.key` and prints the public key.

### 2. Register the Public Key

Add the public key to your `wrangler.toml` under `LUNAR_PUBLIC_KEYS`:

```json
{ "my-project": "8a7b3c4d5e6f..." }
```

### 3. Sign a JWT

Use the private key to sign a JWT with the standard header and payload including the `sub` (project name) claim. Pass it in the `Authorization: Bearer <token>` header when calling `/private/lunar-map.md`.

## Configuration

### Environment Variables

| Variable | Description | Default |
|:---|:---|:---|
| `LUNAR_PUBLIC_KEYS` | JSON map of project names to Ed25519 public keys (hex) | — |
| `LUNAR_CLOCK_SKEW_SECONDS` | Allowed clock skew for JWT validation | `30` |
| `LUNAR_KEY_CACHE_TTL_SECONDS` | Public key cache duration | `300` |
| `LUNAR_GATEWAY_BUFFER_LIMIT_BYTES` | Memory limit before streaming (2MB default) | `2097152` |

## Relationship with Other Components

| Component | Relationship |
|:---|:---|
| `lunar` CLI | Produces `lunar-map.json` consumed by lunar-gateway |
| `lunar-serve` | Local development alternative to lunar-gateway |
| `lunar-scope` | Can consume `lunar-map.json` from lunar-gateway |
| Ecosystem Mother Spec | Defines the architecture in Section 8 |

`lunar-gateway` is a read-only data distributor. It never writes to disk or modifies configuration files.
